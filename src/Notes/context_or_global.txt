We use the **Context API** in React to **share data globally across components without passing props manually at every level (prop drilling).**

Let’s break it down simply 👇

---

## 🚨 Problem (Without Context API)

Imagine you have this structure:

```
App → Parent → Child → Grandchild
```

Now suppose you want to send **user data** from `App` to `Grandchild`.

👉 You have to pass props like this:

```
App → Parent → Child → Grandchild
```

Even if `Parent` and `Child` don’t need it, they still have to pass it.

This is called **prop drilling** ❌
It makes code messy and hard to maintain.

---

## ✅ Solution (Context API)

The **Context API** lets you **store data in one place and access it anywhere** in the component tree.

👉 No need to pass props manually through every level.

---

## 🧠 What Context API Does

* Creates a **global state**
* Allows any component to:

  * **read data**
  * **update data**
* Avoids unnecessary prop passing

---

## 🔧 Real Example

### 1. Create Context

```js
import { createContext } from "react";

export const UserContext = createContext();
```

---

### 2. Provide Context (Wrap your app)

```js
<UserContext.Provider value={{ name: "Sai" }}>
  <App />
</UserContext.Provider>
```

---

### 3. Consume Context (Anywhere)

```js
import { useContext } from "react";
import { UserContext } from "./UserContext";

function Profile() {
  const user = useContext(UserContext);
  return <h1>{user.name}</h1>;
}
```

---

## 🎯 When to Use Context API

Use it when data is needed **in many components**, like:

* 👤 User login info
* 🌙 Dark/light theme
* 🌍 Language settings
* 🛒 Shopping cart

---

## ⚠️ When NOT to Use

* Small apps with simple props
* Data used by only 1–2 components

👉 Overusing it can make debugging harder.

---

## 🔥 Simple Analogy

Think of Context API like **WiFi** 📶
Instead of connecting cables (props) to every device,
everyone connects directly to WiFi (context).

Yes — you **can use Context API with APIs**, but one thing first:

👉 Context API is **NOT for storing APIs**,
👉 It’s for **storing and sharing API *data*** across your app.

---

## 🧠 What you actually do

* Call API (fetch/axios)
* Store response in Context
* Use that data anywhere

---

## ✅ Step-by-step (Real Example)

Let’s say you want to fetch users from an API and use it everywhere.

---

### 1. Create Context

```js
import { createContext } from "react";

export const UserContext = createContext();
```

---

### 2. Create Provider + API Call

```js
import { useState, useEffect } from "react";
import { UserContext } from "./UserContext";

function UserProvider({ children }) {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []);

  return (
    <UserContext.Provider value={{ users }}>
      {children}
    </UserContext.Provider>
  );
}

export default UserProvider;
```

---

### 3. Wrap Your App

```js
import UserProvider from "./UserProvider";

function App() {
  return (
    <UserProvider>
      <YourRoutes />
    </UserProvider>
  );
}
```

---

### 4. Use Data Anywhere

```js
import { useContext } from "react";
import { UserContext } from "./UserContext";

function UsersList() {
  const { users } = useContext(UserContext);

  return (
    <div>
      {users.map(user => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}
```

---

## 🔥 What’s happening here

* API is called **once**
* Data stored in Context
* Any component can access it 🚀

---

## ⚡ Advanced (Best Practice)

Add loading + error handling:

```js
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  fetch("API_URL")
    .then(res => res.json())
    .then(data => {
      setUsers(data);
      setLoading(false);
    })
    .catch(err => {
      setError(err);
      setLoading(false);
    });
}, []);
```

# ✅ Structure (What we will build)

* **UserContext** → handles users API
* **ProductContext** → handles products API
* Wrap both in App
* Use anywhere

---

# 🔹 1. Create User Context

```js
// UserContext.js
import { createContext, useState, useEffect } from "react";

export const UserContext = createContext();

export function UserProvider({ children }) {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []);

  return (
    <UserContext.Provider value={{ users }}>
      {children}
    </UserContext.Provider>
  );
}
```

---

# 🔹 2. Create Product Context

```js
// ProductContext.js
import { createContext, useState, useEffect } from "react";

export const ProductContext = createContext();

export function ProductProvider({ children }) {
  const [products, setProducts] = useState([]);

  useEffect(() => {
    fetch("https://fakestoreapi.com/products")
      .then(res => res.json())
      .then(data => setProducts(data));
  }, []);

  return (
    <ProductContext.Provider value={{ products }}>
      {children}
    </ProductContext.Provider>
  );
}
```

---

# 🔹 3. Wrap Both in App

👉 Order doesn’t matter much, just wrap properly:

```js
import { UserProvider } from "./UserContext";
import { ProductProvider } from "./ProductContext";

function App() {
  return (
    <UserProvider>
      <ProductProvider>
        <YourRoutes />
      </ProductProvider>
    </UserProvider>
  );
}
```

---

# 🔹 4. Use Them Anywhere

### 👉 Using Users

```js
import { useContext } from "react";
import { UserContext } from "./UserContext";

function Users() {
  const { users } = useContext(UserContext);
  return users.map(u => <p key={u.id}>{u.name}</p>);
}
```

---

### 👉 Using Products

```js
import { useContext } from "react";
import { ProductContext } from "./ProductContext";

function Products() {
  const { products } = useContext(ProductContext);
  return products.map(p => <p key={p.id}>{p.title}</p>);
}
```

---

# 🔥 Why this is better

* Clean separation ✅
* Easy to maintain ✅
* Scales well for big apps ✅
* No unnecessary re-renders (important!) ✅

---

# ⚡ Pro Tip (Very Important)

If both contexts are used together often, you can create a **CombinedProvider**:

```js
function AppProviders({ children }) {
  return (
    <UserProvider>
      <ProductProvider>
        {children}
      </ProductProvider>
    </UserProvider>
  );
}
```

Then in App:

```js
<AppProviders>
  <YourRoutes />
</AppProviders>
```

# 🧠 What is `children`?

👉 `children` is a **special prop in React**
It represents **whatever you put inside a component**

---

## 🔍 Example

```js
<ProductProvider>
  <App />
</ProductProvider>
```

👉 Here:

```js
<App />
```

is the **`children`**

---

## 💡 So inside your provider:

```js
function ProductProvider({ children }) {
```

👉 means:

> “Give me whatever is inside this component”

---

# 🎯 Why we wrap with `<ProductContext.Provider>`?

```js
<ProductContext.Provider value={{ products }}>
  {children}
</ProductContext.Provider>
```

👉 This means:

> “All components inside `children` can access `products` globally”

---

## 🔥 Visual Understanding

```
ProductProvider
   ↓
ProductContext.Provider (data available here)
   ↓
App
   ↓
All components inside App
```

👉 So:

* App ✅ can access context
* All child components ✅ can access context

---

# ❌ What if you don’t use `{children}`?

```js
<ProductContext.Provider value={{ products }}>
</ProductContext.Provider>
```

👉 Then NOTHING is inside it 😅
👉 No component can access your data

---

# 🧠 Simple Analogy

Think of it like a **WiFi Router** 📶

* `<ProductContext.Provider>` = WiFi Router
* `value={{ products }}` = Internet data
* `{children}` = Devices connected

👉 Only devices inside range (children) get internet

---

# ⚡ One More Example

```js
function Box({ children }) {
  return <div>{children}</div>;
}

<Box>
  <p>Hello</p>
</Box>
```

👉 Output:

```
Hello
```

Because `<p>Hello</p>` is the **children**

## ✅ Short Answer

👉 You should wrap your Context **at the highest level where all components need access**
👉 Usually that is **inside `App.js` (around your routes)**

---

# 🎯 Best Practice

### ✔️ Wrap around routes

```js
import { BrowserRouter } from "react-router-dom";
import { UserProvider } from "./UserContext";
import { ProductProvider } from "./ProductContext";
import AppRoutes from "./routes";

function App() {
  return (
    <BrowserRouter>
      <UserProvider>
        <ProductProvider>
          <AppRoutes />
        </ProductProvider>
      </UserProvider>
    </BrowserRouter>
  );
}
```

👉 Now:

* All pages (Home, Profile, Dashboard, etc.) ✅
* All components inside routes ✅
  can access your context

---

# 🧠 Why here?

Because:

```
App
 └── Routes
      └── Pages
           └── Components
```

👉 Wrapping at top ensures **everything below gets access**

---

# ❌ What if you wrap inside a single page?

```js
function Home() {
  return (
    <ProductProvider>
      <HomeContent />
    </ProductProvider>
  );
}
```

👉 Then ONLY `HomeContent` gets access
👉 Other pages ❌ cannot access it

---

# ⚡ Rule to remember

👉 Wrap Context:

* 🔹 **Globally needed data → Wrap in App**
* 🔹 **Page-specific data → Wrap in that page only**

---

# 🔥 Real Examples

### Wrap in App (Global)

* User login data ✅
* Theme (dark/light) ✅
* Cart data ✅

### Wrap in Page (Local)

* Form data ✅
* Single page filters ✅

# ✅ 3. Create Routes File

## 👉 `routes.js`

```js
import { Routes, Route } from "react-router-dom";
import Home from "./pages/Home";
import Users from "./pages/Users";
import Products from "./pages/Products";
import NotFound from "./pages/NotFound";

function AppRoutes() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/users" element={<Users />} />
      <Route path="/products" element={<Products />} />
      <Route path="*" element={<NotFound />} />
    </Routes>
  );
}

export default AppRoutes;
```
